## 搬瓦工的"快照"和"迁移"是两回事

先说清楚两个概念，很多人容易混。

**快照（Snapshot）**：是对当前 VPS 状态做一次完整的镜像备份，包含所有文件和已安装的软件。可以用来备份、恢复，也可以导出到另一台 VPS 上——这就是快照迁移，也就是"把一台 VPS 的数据搬到另一台上"。

**机房迁移（Migrate to another DC）**：是把你现有的 VPS 从一个数据中心切换到另一个数据中心，数据全部保留，IP 会变，机房位置变。这个操作是搬瓦工的特色功能，大多数商家是没有的。

两件事都会触发 KiwiVM 面板锁定，操作期间你在 KiwiVM 里什么都干不了。

## 搬瓦工迁移快照进度在哪查询

### 机房迁移的进度查询

机房迁移发起后，KiwiVM 首页会显示这样一段信息：


User-initiated maintenance task is currently in progress.
KiwiVM will remain locked until the task completes.
Please try again in a few minutes. (788888)
Additional information: Migration from v1234@JPOS_1 to v5678@USCA_6: 0% complete [ 0 bytes out of 20.00 GB ]


这就是迁移进度。`Additional information` 那行会实时显示：

- 从哪个机房迁到哪个机房
- 当前完成百分比（比如 23% complete）
- 已传输多少数据 / 总数据量

刷新 KiwiVM 页面，这个数字会更新。不需要等邮件，直接看面板就行。

### 快照操作的进度查询

创建快照时，面板会显示 `Snapshot creation in progress`。恢复快照时，显示 `Snapshot restoration in progress`。这两个状态都代表任务还在执行，面板暂时锁定。

不一样的是，快照操作**没有百分比进度**，只有"任务进行中"的提示。完成后你会收到一封邮件通知。嫌等得无聊就盯着邮箱，不然就去干别的，操作完它自然会好。

快照创建时间从几分钟到一个小时不等，主要取决于 VPS 里存了多少数据。数据少的话十来分钟搞定，数据多的耐心等。

## 完整的快照迁移操作流程（VPS A → VPS B）

有些朋友搜"搬瓦工迁移快照查询"，其实是想搞清楚用快照把数据从一台 VPS 搬到另一台的完整流程。下面把步骤理清楚。

**前提条件**：快照只能在同架构之间传输——KVM 套餐之间，或者 OpenVZ 套餐之间。KVM 和 OpenVZ 之间是不互通的。搬瓦工目前主流套餐都是 KVM 架构，基本不用担心这个问题。

**第一步：在 VPS A 上创建快照**

1. 登录 VPS A 的 KiwiVM 控制面板
2. 左侧菜单找到 **Snapshots**，点击进入
3. 点击 **Create new snapshot** 按钮
4. 输入快照备注名（随便写，方便自己识别）
5. 再次点击 **Create new snapshot** 确认
6. 等待快照创建完成（面板提示 `Snapshot creation in progress`，完成后收邮件通知）

**第二步：导出快照，获取 VEID 和 TOKEN**

1. 快照创建完成后，回到 Snapshots 页面
2. 在快照记录旁边找到 **Export** 选项，点击
3. 系统会生成一个 VEID（VPS 编号）和 TOKEN（导出令牌）——把这两个值记下来，马上要用

**第三步：在 VPS B 上导入快照**

1. 登录 VPS B 的 KiwiVM 控制面板
2. 同样进入 **Snapshots** 页面
3. 点击 **Import a snapshot**
4. 填入刚才记录的 VEID 和 TOKEN
5. 点击 **Import** 确认

**第四步：恢复快照**

1. 导入完成后，在 VPS B 的 Snapshots 页面找到刚导入的快照
2. 点击 **Restore**，系统会提示确认
3. 注意：恢复操作会**覆盖 VPS B 上的所有现有数据**，确认没问题再点 `I agree, restore snapshot`
4. 等待恢复完成，完成后收邮件通知

恢复完成，VPS A 的所有数据就完整地搬到 VPS B 了。

## 搬瓦工迁移进度"卡住"了怎么判断

很多时候不是卡住了，是正常在跑。先别急。

**正常情况**：面板锁定，显示进度百分比或者 `in progress` 提示，等就好。机房迁移一般几十分钟以内，快照根据数据量不同时间差异比较大。

**出现错误提示的情况**，几个常见报错对照：

| 错误代码 | 提示内容 | 原因 | 处理方式 |
|---|---|---|---|
| 734104 | Region is full | 目标机房资源满了 | 等几小时再试 |
| 734105 | New location has no available IP addresses | 目标机房无可用 IP | 等待后重试或换个机房 |
| 734151 | Migration system is currently not available | 迁移系统临时不可用 | 等 15-30 分钟再试 |
| 734152 | Migration backend is currently not available for this VPS | VPS IP 可能被封，或系统故障 | 先检查 IP 状态 |
| 3009005 | KiwiVM control panel is temporarily unavailable | 面板维护中 | 等几分钟刷新 |

734151、734104、3009005 这几个都是临时性的，等一等基本自己就好。

734152 稍微复杂一点。这个报错最常见的原因是 VPS 的 IP 被封锁了，KiwiVM 后端没法和 VPS 正常通信，所以迁移命令下不去。检查方式：把链接里的 `VPS_ID` 替换成你自己的 VPS ID，访问搬瓦工官方的 IP 检测页面，点 **Test Main IP** 按钮。返回 `IP NOT BLOCKED` 就正常；`IP BLOCKED` 就说明 IP 被墙了，先处理 IP 问题再说迁移。

## 几个搬瓦工快照的细节，坑踩过才知道

**快照默认 30 天后自动删除。** 这是我第一次用快照没意识到的，以为存着就永远在。实际上 30 天到期，快照就没了。如果是重要备份，记得手动下载到本地，或者设置为 Sticky Snapshot。

**Sticky Snapshot（粘滞快照）不会过期。** 但最多只能设置 2 个，超出后就要手动管理。做大版本迭代前可以用这个功能留一个永久存档。

**KVM 和 KVM 之间才能互导。** 这条前面提了，再强调一遍。搬瓦工目前主要套餐（CN2 套餐、CN2 GIA 系列、香港套餐等）都是 KVM 架构，相互之间都可以用快照迁移数据，不用担心兼容性。

**创建快照时 VPS 会短暂重启。** 官方说明是停机时间只有几秒，不过确实会重启。如果你的 VPS 上跑着对中断敏感的服务，最好选一个访问量少的时间段操作。

## 快照迁移 vs 机房迁移：什么场景用哪个

| 场景 | 推荐方式 |
|---|---|
| 想换个更好的机房，数据全搬过去 | 直接用 KiwiVM 的机房迁移功能（Migrate to another DC） |
| 买了新套餐，要把老 VPS 数据复制过来 | 用快照迁移（创建 → 导出 → 导入 → 恢复） |
| 对现有系统做备份，怕出问题能回滚 | 创建快照保存，设为 Sticky 防止过期 |
| 定期自动备份 | 用 KiwiVM 自带的免费自动备份功能（Backups），不用手动操作 |

顺便提一嘴：搬瓦工的 KiwiVM 同时提供快照和自动备份两个免费功能。快照是手动触发的，自己控制时间点；自动备份是系统按周期自动跑的，备份间隔取决于 VPS 数据量，两者可以配合使用。

## FAQ：搬瓦工迁移快照常见问题

**Q：迁移机房之后数据会丢失吗？**

不会。机房迁移是把整个 VPS 数据完整搬到目标机房，所有文件、软件、配置都在，唯一变的是 IP 地址。

**Q：快照创建到一半失败了怎么办？**

重新创建一次就行。如果反复失败，检查一下 VPS 的硬盘剩余空间，快照需要足够的空间才能写入。

**Q：机房迁移需要多长时间？**

取决于 VPS 上数据量大小。20 GB 的盘通常在几十分钟内完成，数据越少越快。迁移过程中 VPS 服务正常运行，只有 KiwiVM 面板会锁定。

**Q：快照能不能在不同搬瓦工套餐之间互导？**

可以，只要两台都是 KVM 架构就行。搬瓦工目前所有主流套餐（CN2 系列、CN2 GIA 系列、香港套餐等）都是 KVM，互相之间导快照没有问题。

**Q：迁移进度一直显示 0%，是不是卡死了？**

不一定。0% 有时候是刚开始，传输还没启动。等个几分钟刷新再看。如果超过半小时还是 0%，可能是目标机房资源问题，参考前面的错误代码处理思路，稍后重试或者联系客服。
